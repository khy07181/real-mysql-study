## 문제

### Q1. ACID 속성 (현서, 주호)

트랜잭션의 ACID 속성 각각을 설명하고, MySQL InnoDB에서 어떻게 보장하는지 설명하세요.

### Q2. 트랜잭션 격리 수준 (현서, 영섭)

MySQL의 4가지 트랜잭션 격리 수준을 나열하고, 각각에서 발생할 수 있는 문제점을 설명하세요.

### Q3. REPEATABLE READ (영섭, 지훈)

MySQL InnoDB의 기본 격리 수준인 REPEATABLE READ에서 Phantom Read가 발생하지 않는 이유를 설명하세요.

### Q4. InnoDB 잠금 종류 (지훈, 연호)

InnoDB의 레코드 락, 갭 락, 넥스트 키 락의 차이점을 설명하세요.

### Q5. 잠금 대기와 타임아웃 (연호, 진영)

잠금 대기 시간이 초과되면 어떤 일이 발생하나요? 관련 설정은 무엇인가요?

### Q6. 데드락 (진영, 하영)

데드락이란 무엇이며, MySQL에서 데드락이 발생하면 어떻게 처리되나요?

### Q7. 네임드 락과 메타데이터 락 (하영, 주호)

네임드 락(Named Lock)과 메타데이터 락(Metadata Lock)의 용도를 각각 설명하세요.

### Q8. 실무/면접 질문

"트랜잭션에서 SELECT ... FOR UPDATE와 SELECT ... LOCK IN SHARE MODE의 차이점을 설명해주세요. 각각 언제 사용하나요?"
