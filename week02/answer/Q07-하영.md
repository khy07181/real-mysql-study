## 네임드 락(Named Lock)과 메타데이터 락(Metadata Lock)의 용도를 각각 설명하세요.

### Named Lock
- 임의의 문자열 이름을 기준으로 획득하는 사용자 정의 잠금이다.
- 테이블·행과는 전혀 무관하며, 범위가 서버 전체로 적용된다.

용도
- 애플리케이션 레벨 동기화를 위해 사용된다.
- 동시에 시작, 진행, 완료되지 않아야 하는 작업에 주로 사용된다.
    - 하루에 한 번 실행되어야 하는 배치
    - 파일 핸들링

### Metadata Lock
- 테이블, 뷰, 인덱스 같은 스키마 객체의 구조를 보호하기 위한 내부 락이다.
- 사용자가 직접 제어하지 않으며, 모든 SQL 실행 시 자동으로 획득된다.

용도
- 스키마 객체의 일관성을 위해 주로 사용된다.
    - SELECT 중 DROP TABLE 방지
    - ALTER TABLE 중 INSERT/UPDATE 방지
